## macos-defaults expansion: .macos pattern adoption

Lean into [mathiasbynens/.macos][BYNENS] more deliberately. The existing
`scripts/configure/macos-defaults.sh` is structurally the same approach
(sectioned shell script of `defaults write` calls, idempotent), with two
local improvements kept: the `set_and_show` verification helper and the
`scripts/configure/` location with `${SCRIPT_DIR}` / `env.sh` sourcing.
What we adopted from Mathias this pass is the *technique* (preamble guard
and the discipline of breadth), not the content.

### Preamble guard

Two `osascript` calls at the top close System Settings (Ventura+) and the
older System Preferences app. Without this guard, an open Settings pane
caches the current preference values in memory and writes them back over
our changes when its window closes, silently undoing the script's work.
Both calls are best-effort and silent if the app is not open.

### Three new delta sections

Captured from this Mac's live state (read via `defaults read`):

- `configure:disk-stores`: `DSDontWriteNetworkStores`, `DSDontWriteUSBStores`
  (both true). Keeps shared volumes and USB sticks free of `.DS_Store`.
- `configure:dock` extension: `tilesize=36`, `mru-spaces=false`,
  `launchanim=false`, `expose-animation-duration=0.1`,
  `expose-group-apps=false`. Compact right-side dock, no auto-rearrange,
  faster Mission Control transitions, one tile per window in Exposé.
- `configure:typing-code`: `ApplePressAndHoldEnabled=false` and every
  `NSAutomatic*` substitution off (capitalization, dashes, periods,
  quotes, spelling correction, text completion). Press-and-hold off
  is essential for vim and code editors (held key repeats vs. accent
  picker).

A fourth set (`chflags nohidden ~/Library` + `xattr -d com.apple.FinderInfo`)
was added on user request, even though Apple hides `~/Library` by default
since 10.7 Lion. Apple uses two mechanisms: the BSD `hidden` chflag AND the
`kIsInvisible` bit inside `com.apple.FinderInfo`; the script clears both.
`~/Documents` and `~/Desktop` look hidden in Finder for a different reason:
iCloud Drive's "Desktop & Documents Folders" sync sets a
`com.apple.fileprovider.detached#B` xattr that re-routes the Finder display
through iCloud Drive. Disabling that requires the System Settings toggle
under Apple ID → iCloud → Drive; out of scope for this script.

Script re-ran clean and idempotent; every new line echoed back the
expected value, and `xattr ~/Library` now shows only `com.apple.provenance`.

## Terminal-first editor stack: helix + neovim + glow

Exploring whether terminal-first editing can absorb the "see what changed
in gitui, drop notes for Claude, hand back" loop without bouncing to
VS Code. Already love gitui; the missing piece is a fast TUI editor.

### Decision context

Helix is Rust-native and batteries-included (LSP, fuzzy picker,
multi-cursor built in) but has no plugin system. The Steel plugin work is
still PR #8675 against helix-editor/helix, and the underlying `conceal`
feature that would enable render-markdown-style inline rendering is
issue #5798 (open since 2024, rated hard). Helix means accepting markdown
as syntax-highlight only, with `glow` in a tmux/zellij split for preview.

Neovim with the LazyVim distro plus `render-markdown.nvim` gives Obsidian-
style inline rendering (headings with backgrounds, checkboxes as glyphs,
code-block backgrounds, tables aligned) in the same buffer you're editing.
Trade is the vim model, already familiar but not loved.

Picked LazyVim over kickstart.nvim because the goal is to *use* neovim,
not to *learn* it. Vim familiarity is already there, render-markdown is
the actual draw, and folke's curation handles the rough edges in 10
minutes versus a weekend of wiring.

### Brewfile additions

```ruby
brew "helix"
brew "neovim"
brew "glow"     # markdown viewer for the Helix-in-a-split workflow
```

### scripts/configure/nvim.sh

Idempotent configure script: clones the LazyVim starter to
`~/.config/nvim` (detaching from upstream so the config is owned locally),
then copies the `render-markdown.nvim` plugin spec from
`manifests/nvim/lua/plugins/markdown.lua` into place. Re-running yields
skip status on both steps.

Mirrors the existing `claude-settings.sh` pattern: data lives in
`manifests/`, scripts orchestrate. The plugin spec is a tracked file
diffable on its own, no heredoc'd content inside the shell script.

### Next sanity check

First `nvim` launch triggers lazy.nvim's bootstrap, installs all LazyVim
defaults plus `render-markdown.nvim`. Run `:checkhealth` after to confirm
language servers and tree-sitter parsers settled.

For Helix, `hx --health` shows installed language servers and grammar
status. No further config needed to start.
