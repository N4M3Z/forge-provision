# 2026-05-17 — revdiff + opencode integration, skills/DotfilesStrategy

## Terminal-first review of AI sessions

Picked [revdiff][REVDIFF] (Umputun, v1.3.0) over [tuicr][TUICR] for the
in-session review loop after a brainstorming pass with three parallel
research agents (editor plugins, standalone TUIs, pre-PR diff tools).

Reasoning: revdiff is purpose-built for the workflow of reading what an AI
assistant just changed, dropping inline annotations, and piping them back
to the assistant as the next prompt. It has dedicated plugins for both
Claude Code and OpenCode. tuicr is the stronger choice if GitHub PR review
is the dominant mode, but for this Mac the immediate need is local /
pre-commit review of Claude sessions, not async PR review. tuicr stays
on the radar for when GitHub PR review becomes the bottleneck.

Other tools considered and ranked lower:

- [danobi/prr][PRR] — file-edit metaphor, superseded by revdiff/tuicr TUIs
- [dlvhdr/gh-dash][GHDASH] + [gh-review][GHREVIEW] — two-tool stitching
- [agynio/gh-pr-review][GHPRREVIEW], [yukikotani231/gh-pr-review][YUK],
  [k1LoW/gh-pr-reviews][K1L1], [k1LoW/gh-copilot-review][K1L2] — `gh`
  extensions, narrower scope
- [pwntester/octo.nvim][OCTO] — strong nvim-native option, kept as fallback
- [ushironoko/octorus][OCTORUS] — heavier Rust TUI, AI-rally mode
- [jj-vcs/jj][JJ] — workflow paradigm shift, not a tool install

## Brewfile + configure script

```ruby
tap "umputun/apps"
brew "umputun/apps/revdiff"
```

`brew bundle install` pulled revdiff 1.3.0 from the tap. Binary at
`/opt/homebrew/bin/revdiff`, `revdiff --version` confirms.

New `scripts/configure/revdiff.sh` wires the opencode plan-review plugin.
The upstream `plugins/opencode/setup.sh` copies five files into
`~/.config/opencode/{commands,tools,plugins}` and registers the plugin
entry in `opencode.json`. To stay aligned with whatever the maintainer
ships next, the configure script delegates to upstream's setup.sh rather
than reimplementing the copy logic locally:

1. Clones revdiff to `/tmp/revdiff-opencode-XXXXXX` (shallow, depth 1)
2. Runs `bash plugins/opencode/setup.sh`
3. Cleans up via `trap` on exit
4. Idempotent guard: skips if
   `~/.config/opencode/plugins/launch-plan-review.sh` already exists

First run wrote five files into opencode's config and appended
`./plugins/revdiff-plan-review.ts` to the plugin array in `opencode.json`.
The pre-existing provider entries (proton-lumo, lmstudio-local) were
preserved by setup.sh's append-not-replace approach. Re-run gave
`skip:revdiff (opencode plan-review plugin already wired)`.

## Pending follow-up: Claude Code plugin

The Claude Code integration is a runtime install inside Claude Code itself,
not provisionable from a shell script. Run once:

```
/plugin install revdiff@revdiff
```

Inside Claude Code, after the next session restart. After that, Claude
Code can launch revdiff as a terminal overlay (Ghostty supports the
overlay mechanism; tmux/zellij/kitty/wezterm are the other supported
hosts).
