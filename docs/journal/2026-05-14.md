### enhancd retirement → zoxide via Prezto module

Decision: drop enhancd, use the dotzsh-contrib `zoxide` module. Reasoning:
the `--cmd cd` override path zoxide offers caused subtle Claude Code Bash
issues in a past session. Module path keeps `cd` as the unmodified zsh
builtin and gives us `z <query>` and `zi <query>` (the latter via fzf).

- `dotfiles/zpreztorc`: removed `'enhancd'` from pmodule list, removed the
  three `:prezto:module:enhancd` zstyle config lines + their comment header.
  Added `'zoxide'` to the pmodule list at line 75 right after `'completion'`.
- `dotfiles/zshrc`: removed the direct `eval "$(zoxide init --cmd cd zsh)"`.
- `~/.zsh/runcoms/zpreztorc` (dotzsh fork): same three enhancd zstyles
  removed. Uncommitted.
- `~/.zsh/contrib/` (dotzsh-contrib): `git submodule deinit -f enhancd/external`
  then `git rm -rf enhancd`. Four deletions staged; submodule entry removed
  from `contrib/.gitmodules`. **Detached HEAD** — needs `git switch -c
  drop-enhancd` before committing.
- Brewfile gained `brew "zoxide"` and `brew "fzf"` (zoxide installs as 0.9.9
  bottle; fzf 0.72.0 backs `zi`'s picker). fzy was considered but skipped —
  fzf is the de facto standard for everything else (vim plugins, completions),
  and at zoxide-sized databases fzy's speed advantage is sub-millisecond.

The dotzsh-contrib `zoxide` module is a 5-line wrapper with **no zstyles**.
Upstream zoxide is configured via env vars (`_ZO_CMD_PREFIX`,
`_ZO_DATA_DIR`, `_ZO_EXCLUDE_DIRS`, `_ZO_FZF_OPTS`, `_ZO_MAXAGE`,
`_ZO_RESOLVE_SYMLINKS`, `_ZO_ECHO`). If we want enhancd-parity ergonomics
later, patch `contrib/zoxide/init.zsh` to read `:prezto:module:zoxide *`
zstyles and turn them into env vars before the eval. Not blocking.

The previous dotzsh-contrib `enhancd` module documented two zstyles (`filter`
fed `ENHANCD_FILTER`, `command` fed `ENHANCD_COMMAND`); `show-fullpath` was a
zpreztorc line with no plumbing — dead config. Documentation history
preserved by `git -C ~/.zsh/contrib show HEAD:enhancd/README.md` if anyone
needs to reconstruct.

### atuin Up-arrow tuning

Goal: make Up-arrow feel closer to zsh-history-substring-search (type prefix,
press Up, prompt expands inline with the last match). Atuin always opens a
TUI, but `inline_height_shell_up_key_binding = 1` shrinks it to one line
right above the prompt — visually it sits where the expansion would have
appeared. Full TUI still available on Ctrl-R with `inline_height = 25`.

`dotfiles/config/atuin/config.toml` additions:

```toml
style = "compact"
inline_height = 25
inline_height_shell_up_key_binding = 1
```

If the one-line popup still feels intrusive after live use, the fallback is
reverting Up-arrow to plain `zsh-history-substring-search` (already loaded
by Prezto via `'history-substring-search'` in pmodule list) and keeping atuin
on Ctrl-R only. That's a one-line toggle in atuin config:
`search_mode_shell_up_key_binding = "disabled"`.
