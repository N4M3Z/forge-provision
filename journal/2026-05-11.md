## Research notes captured here for the next iteration

- **No official Anthropic export/import** for Claude Code chat history. Issue [anthropics/claude-code#18645](https://github.com/anthropics/claude-code/issues/18645) closed as duplicate.
- Claude Code 2.1.9+ blocks cross-machine `/resume` — transcripts remain as `.jsonl` files (readable, greppable) but the UI may show "No conversation found to continue". Migrating the data is still worth it for grep-back + memory continuity.
- Third-party tools considered: [`miwidot/ccms`](https://github.com/miwidot/ccms) (rsync-over-SSH, full `.claude/` sync), [`porkchop/claude-code-sync`](https://github.com/porkchop/claude-code-sync) (git-based). Both overkill for one-shot migration; we use plain rsync.
- Gogh has no Homebrew formula and doesn't support default Terminal.app on macOS (iTerm only). Replaced by direct `.terminal` file authoring from `mbadolato/iTerm2-Color-Schemes`.
- NSFont archive encoding (used inside `.terminal` plist for the profile's font) is binary plist; not shell-friendly. Font selection for the N4M3Z profile is a one-time GUI action in Terminal Settings; future themer-based regeneration will bake it in.
